Key Entity Relationships

EntityRelationships
UserHas one Cart (1:1), one Library (1:1), many WishlistItems (1:N), many Reviews (1:N), sent/received Messages, sent/received FriendRequests, many Orders, many Notifications, optional DeveloperProfile (1:1), many Reports, many BanLogs, many Payouts, many SupportTickets
GameBelongs to a Developer (User via DeveloperProfile), has many Categories (N:M via GameCategory), many CartItems, many LibraryGames, many OrderItems, many Reviews, many WishlistItems
CartBelongs to one User (1:1), contains many CartItems (1:N) → each CartItem points to one Game
LibraryBelongs to one User (1:1), contains many LibraryGames (1:N) → each LibraryGame points to one Game
OrderBelongs to one User (1:N), contains many OrderItems (1:N) → each OrderItem points to one Game with price-at-purchase and commission
SupportTicketBelongs to one User (optional, 1:N), contains many TicketReplies (1:N)
FriendshipLinks two Users (Requester + Receiver) with status (Pending / Accepted / Rejected)
MessageSent by one User to another User with read tracking
AuditLogTracks entity changes across the system for compliance

2. Stakeholder Personas

Customer (Omar)

Role: End-user who browses, purchases, and plays games.

Goals: Discover new games, buy securely, build library, connect with friends.

Pain Points: Unclear refund policy, difficulty finding specific games, payment issues.

Developer (Sara)

Role: Game creator who uploads and sells games on the platform.

Goals: Publish games easily, track sales analytics, receive timely payouts.

Pain Points: Complex upload process, delayed payout cycles, limited visibility into sales data.

Admin (Ahmed)

Role: Platform manager responsible for moderation, user management, and system health.

Goals: Efficient content moderation, accurate revenue reporting, fraud detection.

Pain Points: Fake or inappropriate game submissions, fraudulent transactions, manual approval bottlenecks.

3. Role & Permission Matrix

CUSTOMER
  • Browse and search game catalog
  • View game details and reviews
  • Purchase games via Stripe or store credit
  • Write, edit, and delete reviews
  • Add/remove games from wishlist
  • View owned games in library
  • Download purchased games
  • Send/receive friend requests
  • Chat with friends (SignalR)
  • View and manage notifications
  • Submit and track support tickets
  • Apply coupons at checkout
  • Request refunds
  • Edit profile, change password
  • Post on profiles
DEVELOPER
  • All CUSTOMER permissions
  • Create developer profile (studio)
  • Upload and manage game listings
  • Edit game metadata, media, files
  • Manage game versions and builds
  • Set system requirements per game
  • View developer dashboard with analytics
  • Track downloads, revenue, average rating
  • View game-specific reviews
  • Request sales/discounts (admin approval)
  • Track payout history and pending balances
  • View developer-specific support tickets
ADMIN
  • All permissions across platform
  • Dashboard with revenue, orders, user stats
  • CRUD management of all games
  • CRUD management of all users (change roles, delete)
  • CRUD management of categories
  • View and manage all orders
  • Approve/reject developer applications
  • Approve/reject sale requests
  • Moderate reviews (delete inappropriate)
  • Manage support tickets (reply, update status, delete)
  • Manage refund requests
  • Manage developer profiles (demote, reactivate, delete)
  • View platform reports and audit logs
  • View system settings
